Goal: Build an AI system that helps developers understand unfamiliar codebases through repository analysis, architecture understanding, and intelligent Q&A.

Why am I building this?
Learn AI systems
Learn software architecture
Learn retrieval systems
Learn agentic workflows
Build something genuinely useful


Q1 Who is this product for?
-> It’s for students and new beginner developers that struggle to understand the codebase looking at it.

Q2 What is the biggest frustration they face when entering a new repository?
-> When looking at it, understanding what is where and doing what is time consuming. And if new things are to be added, whole architecture needs to be studied so nothing breaks.

Q3 Describe the dream outcome. Imagine our product is perfect. A developer uploads a repo. What happens next?
-> So what this will do is, when this project is complete like all modules and everything then it would be able to generate visual graphs, dependency graphs, whole readme files, system architecture, code analysis and many other things. 

Q4 What should this product NEVER try to do?
-> This should never try to change files on its own unless asked or unless connected to another agent.

Q5 why do i personally want to build this?
-> For myself to learn how to actually build a project in a systematic way.
